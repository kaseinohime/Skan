## Step 1: プロジェクト初期化

### 目的
Next.js + Supabase の開発環境をセットアップし、基本構造を整える。

### 作業内容

1. **Next.js プロジェクト作成**
   ```bash
   pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir=false
   ```

2. **依存パッケージのインストール**
   ```bash
   pnpm add @supabase/supabase-js @supabase/ssr
   pnpm add lucide-react
   pnpm add zod react-hook-form @hookform/resolvers
   pnpm add date-fns
   pnpm add openai
   pnpm add -D prettier eslint-config-prettier
   ```

3. **shadcn/ui の初期化**
   ```bash
   pnpm dlx shadcn@latest init
   ```
   基本コンポーネントの追加:
   ```bash
   pnpm dlx shadcn@latest add button card input label textarea select badge dialog alert-dialog dropdown-menu sheet tabs toast sonner avatar separator skeleton table calendar popover command
   ```

4. **ディレクトリ構造の作成**
   ```
   app/(auth)/, app/(dashboard)/, app/shared/, app/api/
   components/ui/, components/layout/, components/calendar/, components/posts/, components/preview/, components/approval/
   lib/supabase/, lib/validations/
   types/
   hooks/
   supabase/migrations/
   ```

5. **Supabase クライアントの設定**
   - `lib/supabase/client.ts` - ブラウザ用
   - `lib/supabase/server.ts` - Server Components / Route Handlers 用
   - `lib/supabase/middleware.ts` - Middleware 用

6. **環境変数ファイルの作成**
   - `.env.local` に Supabase URL / Key / OpenAI Key を設定

7. **TypeScript の厳格設定**
   - `strict: true` を確認

8. **Prettier / ESLint の設定**

### 完了条件
- [×] `pnpm dev` でローカルサーバーが起動する
- [ ] shadcn/ui のボタンがレンダリングされる
- [ ] Supabase クライアントが正しく初期化される
- [ ] ディレクトリ構造が TECH_STACK.md に沿っている

## Step 2: 認証基盤（Supabase Auth + ロール管理）

### 目的
ユーザー認証とロールベースのアクセス制御の基盤を構築する。

### 作業内容

1. **Supabase でDBマイグレーション作成**
   - `users` テーブル（`auth.users` 拡張）
   - `updated_at` 自動更新トリガー

2. **認証ページの実装**
   - `/login` - メールアドレス + パスワードログイン
   - `/register` - 招待リンク経由の登録（パスワード設定）
   - `/forgot-password` - パスワードリセット

3. **Middleware の実装**
   - 未認証ユーザーを `/login` にリダイレクト
   - ロールに応じたルート保護
   - マスター: `/master/*`
   - 企業管理者: `/dashboard`, `/clients/*`, `/staff/*`, `/settings/*`
   - スタッフ: `/clients/[clientId]/*`
   - クライアント: `/approval`, `/clients/[clientId]/*`（閲覧のみ）

4. **RLS ポリシーの設定**
   - `users` テーブルの RLS

5. **認証ヘルパー関数**
   - `getCurrentUser()` - 現在のユーザー取得
   - `getUserRole()` - ユーザーロール取得
   - `requireAuth()` - 認証必須チェック
   - `requireRole(role)` - ロール必須チェック

6. **Auth Callback ルート**
   - `/api/auth/callback` - メール確認・招待リンクのコールバック

### 完了条件
- [×] ログイン / ログアウトが動作する
- [×] 招待リンク経由で新規ユーザーがパスワードを設定できる
- [×] ロールに応じたルート保護が機能する
- [×] 未認証時に `/login` にリダイレクトされる

5. **ハッシュタグ提案機能**
