# FRONTEND_GUIDELINES（フロントエンドデザインシステム） - エスカン

## 1. デザイン原則

1. **明快さ**: SNS運用の複雑なワークフローをシンプルに可視化する
2. **効率性**: 最小クリックで目的の操作を完了できるUI
3. **一貫性**: shadcn/uiベースの統一されたコンポーネント体系
4. **信頼感**: ビジネスツールとしての落ち着いた配色とプロフェッショナルな印象

## 5. レスポンシブデザイン

### ブレイクポイント

| 名前 | 値 | 用途 |
|------|-----|------|
| sm | 640px | モバイル横向き |
| md | 768px | タブレット |
| lg | 1024px | デスクトップ（サイドバー表示切替点） |
| xl | 1280px | ワイドスクリーン |
| 2xl | 1536px | フルHDモニター |

### レイアウト方針

- **モバイル (< 1024px)**: サイドバーはハンバーガーメニューに格納。カレンダーは週表示がデフォルト
- **デスクトップ (>= 1024px)**: サイドバー常時表示。カレンダーは月表示がデフォルト
- サイドバー幅: 256px (固定)
- メインコンテンツ最大幅: 1280px

## 6. コンポーネントルール

### shadcn/ui 使用方針

- shadcn/ui のコンポーネントを最優先で使用する
- カスタムコンポーネントを作る前に、既存の shadcn/ui で実現できないか検討
- shadcn/ui のコンポーネントは `components/ui/` に配置（CLIデフォルト）
- プロジェクト固有のコンポーネントは `components/` 直下にドメイン別フォルダで整理

### ボタン

| バリアント | 用途 | 例 |
|-----------|------|-----|
| default | 主要アクション | 投稿作成、保存 |
| secondary | 副次アクション | キャンセル、戻る |
| destructive | 削除・危険な操作 | 投稿削除、アカウント削除 |
| outline | 通常操作 | フィルター、ソート |
| ghost | ナビゲーション・補助 | サイドバーメニュー |
| link | テキストリンク風 | 詳細を見る |

### カード

- 投稿カード: 白背景、1px border、rounded-lg (8px)、shadow-sm
- ステータスバッジ付きで左上に表示
- ホバーで shadow-md に変化

### フォーム

- ラベルは入力欄の上に配置
- 必須項目は `*` マークを赤色で表示
- エラーメッセージは入力欄の直下に赤色テキスト (text-sm)
- バリデーションは Zod + react-hook-form で統一

### テーブル

- shadcn/ui の Table コンポーネントを使用
- ヘッダー行は背景色 `--surface`
- ストライプ表示なし（シンプルなボーダー区切り）
- ソート可能カラムにはアイコン表示

### モーダル / ダイアログ

- shadcn/ui の Dialog を使用
- 確認ダイアログ: AlertDialog
- フルスクリーンモーダルは使用しない（SlideOver パターンを推奨）

## 9. カレンダーUI仕様

### 月表示

- 7列（月〜日）× 5〜6行のグリッドレイアウト
- 各セルに投稿カード（ミニ版）を表示
  - ステータスバッジ（色付きドット）
  - 投稿種別アイコン
  - 投稿タイトル（1行、テキスト省略）
- セル内の投稿が3件以上の場合は「+N件」と表示
- 投稿カードのドラッグ＆ドロップで日付変更

### 週表示

- 7列 × 時間帯行のタイムライングリッド
- 投稿予定時間にカードを配置
- 時間帯ごとの投稿密度が視覚的にわかる

### カレンダーのフィルター

- ステータス別フィルター（チェックボックス）
- 担当者別フィルター
- 企画別フィルター
- 投稿種別フィルター（フィード / リール / ストーリーズ / TikTok）

## 10. ダークモード

- Phase 1 ではダークモード非対応（ライトモードのみ）
- shadcn/ui の CSS変数設計を活用し、将来的な対応を見据えた実装にする
- CSS変数は `hsl()` 形式で定義し、ダークモード追加時に値の差し替えのみで対応可能にする

## 11. アニメーション・トランジション

### 基本方針

- 派手なアニメーションは避け、UIの状態変化を自然に伝える程度に留める
- `transition-all duration-200` を基本とする

### 適用箇所

| 要素 | アニメーション | duration |
|------|--------------|----------|
| ボタンホバー | 背景色変化 | 150ms |
| カードホバー | shadow変化 | 200ms |
| サイドバー開閉 | 横幅 + opacity | 300ms |
| モーダル表示 | scale + opacity | 200ms |
| ステータスバッジ変化 | 背景色フェード | 300ms |
| カレンダーD&D | transform | リアルタイム |
| トースト通知 | slide-in + fade | 300ms |

## 12. 通知・トースト

- shadcn/ui の Sonner（Toast）を使用
- 画面右下に表示
- 自動消去: 5秒
- 種類: success（緑）、error（赤）、warning（黄）、info（青）
